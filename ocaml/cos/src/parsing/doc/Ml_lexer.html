Utilities


val update_loc : Lexing.lexbuf -> string option -> int -> bool -> int -> unit
Update the current location with file name and line number. absolute if false means add line to the current line number, if true, replace it with line entirely

create_hashtable size init creates a hashtable with size buckets with initial contents init

keyword_table associates keywords with their tokens
val initial_string_buffer : bytes
An allocation of 256 bytes
val string_buf : bytes Pervasives.ref
A string buffer reference
val string_index : int Pervasives.ref
The next unused index in the buffer
val reset_string_buffer : unit -> unit
Reset the string buffer contents to the original allocation
val store_string_char : char -> unit
Write a char c into the string buffer at the position indicated by the current string_index, creating more space as neccessary. Increment string_index
val store_string : string -> unit
store_string s writes s into string_buf by way of store_string_char
val store_lexeme : Lexing.lexbuf -> unit
Called from the semantic actions of lexer definitions, Lexing.lexeme lexubf returns the string corresponding to the the matched regular expression. store_lexeme lexbuf stores this string in string_buf by way of store_string
val get_stored_string : unit -> string
Gets the contents of string_buff, from 0 to the current string_index, resets the string_buff back to it's original allocation; does not modify string_index
val comment_start_loc : Ml_location.t list Pervasives.ref
A mutuable list of start locations of comments
val in_comment : unit -> bool
in_comment () evaluates to true between the time a comment has been started and not ended, false at all other times
val comment_list : (string * Ml_location.t) list Pervasives.ref
A list of all comments accumulated during lexing
val add_comment : string * Ml_location.t -> unit
Add a comment to the list
val comments : unit -> (string * Ml_location.t) list
Retrieve the list of comments
val with_comment_buffer : (Lexing.lexbuf -> Ml_location.t) -> Lexing.lexbuf -> string * Ml_location.t
with_comment_buffer comment lexbuf uses the string buffer comment_start_loc and the comment rule
